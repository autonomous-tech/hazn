## THE SCREENING PROCESS (DEEP DIVE)

**Section label:** HOW WE SCREEN

**Headline:**
What "Technically Screened" Actually Means.

**Subheadline:**
Every engineer in our pool completed this evaluation. Here's the methodology.

---

**Stage 1 — Application Review**
We assess work history, domain depth, and project complexity. Generalists and juniors don't progress.

**Stage 2 — Technical Assessment**
A hands-on, timed evaluation covering:
- SQL (multi-table joins, window functions, query optimisation)
- Python (data pipeline design, code quality, OOP, testing)
- Data architecture (warehouse patterns, modelling, partitioning)
- ML/AI (model lifecycle, deployment, LLM API integration — role-dependent)

**Stage 3 — System Design Interview**
A live session with a TalentForge technical evaluator. Engineers walk through a realistic design problem relevant to their domain.

**Stage 4 — Communication & Professional Assessment**
English written and verbal evaluation. We assess clarity, async communication style, and ability to work in distributed team environments.

**Stage 5 — Reference Verification**
We contact prior employers or clients. Every engineer in the pool has at least [TBC] verified references.
