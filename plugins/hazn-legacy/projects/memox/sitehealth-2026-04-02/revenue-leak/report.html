Executive Summary

Foundation exists — attribution doesn't

PostHog is actively installed on memox.io and collecting data. The team has configured 8 custom events, 4 conversion actions, and 2 dashboards with 17 insights including DAU, WAU, retention, and a pageview funnel. This is a solid measurement foundation. However, the demo booking funnel is incomplete (click tracked, completion not), there are no ad platform pixels, no UTM strategy, and no consent mechanism. The result: Memox can see what visitors do, but cannot attribute why they came or measure marketing ROI.

KEY: Demo Booking Completion Is Not Tracked

PostHog tracks clicked_book_demo and calendly_opened — but there is no event for when a visitor actually completes a Calendly booking. This means Memox can see intent but cannot measure actual conversions. Every funnel, every attribution model, every ROI calculation depends on this one missing event.

Critical Gap

No Calendly completion event PostHog API

The funnel captures intent (clicked_book_demo → calendly_opened) but not the outcome. To close this gap, implement one of:

Option A (recommended): Listen for the Calendly calendly.event_scheduled postMessage event in the parent window and fire a PostHog event:
window.addEventListener('message', (e) => { if (e.data.event === 'calendly.event_scheduled') { posthog.capture('calendly_booking_complete', { invitee_uri: e.data.payload?.invitee?.uri }); } });

Option B: Use Calendly webhooks to send a server-side event to PostHog via the Capture API.

Recommendation

Define a PostHog "Key Conversion" action for demo bookings

Once calendly_booking_complete is firing, create a PostHog Action combining the full funnel: CTA click → Calendly open → booking complete. This enables funnel analysis with drop-off rates at each step, directly in the PostHog UI.

Medium Priority

Enable PostHog cookieless mode as immediate step

PostHog supports a persistence: 'memory' configuration that avoids setting cookies entirely. This eliminates the need for a cookie consent banner for analytics while still capturing all events. For a B2B SaaS targeting US equipment dealers, this is a pragmatic compliance approach:

posthog.init('phc_tqW3wuoSEKKiO5AHl3UqQzxtSZ4ZowsIYyJEF1yz2M0', { api_host: 'https://us.i.posthog.com', persistence: 'memory', // no cookies capture_pageview: true, capture_pageleave: true, autocapture: true });
