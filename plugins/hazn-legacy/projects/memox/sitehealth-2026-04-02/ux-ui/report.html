Executive Summary

Clean modern design, accessibility gaps below the surface

The homepage is visually well-structured with clear hierarchy and logical flow. Next.js delivers fast rendering with Cloudflare CDN and font preloading. However, WCAG 2.1 AA compliance has gaps — no skip navigation, limited ARIA landmark roles, no prefers-reduced-motion support — and the user journey from homepage to demo booking has friction: the only conversion path requires committing to a live call.

High Priority

No skip navigation link Observed

WCAG 2.4.1 requires a mechanism to bypass repeated blocks of content. Keyboard-only users must tab through the entire header navigation on every page load. Fix: Add <a href="#main" class="sr-only focus:not-sr-only">Skip to content</a> as the first focusable element.

Medium Priority

Animations have no reduced motion fallback Observed

CSS animation classes (animate-fade-in, animate with animation-delay) are applied to hero elements. No @media (prefers-reduced-motion: reduce) query was found. Users who have requested reduced motion in their OS settings will still see animations. Fix: Add a reduced motion media query that disables or simplifies all transitions.

Medium Priority

Limited ARIA landmark roles Observed

Only aria-expanded (mobile menu) and aria-controls found. Missing: role="main" on main content area, role="navigation" on primary nav (native <nav> may cover this), role="banner" on header. Fix: Add semantic ARIA roles to major page sections, or confirm HTML5 landmark elements (<header>, <main>, <nav>, <footer>) are properly used.

Section 3

Cognitive Load

Score: 7/10 — Well-structured page with progressive disclosure. Some decision overload in pricing.

FAQ accordion pattern reduces cognitive load

9 FAQ items in expandable format let users access specific answers without being overwhelmed. This is the correct pattern for objection handling at the bottom of a long-form landing page.

Three-step setup reduces perceived complexity

"Connect → Configure → Convert" as a three-step process makes implementation feel achievable. The HowTo schema markup reinforces this structure for AI engines.

Medium Priority

Long page without progress indicators Assessment

The homepage is 7+ sections long. On mobile, users have no sense of how much content remains. A progress indicator (scroll bar, section dots, or sticky section labels) would improve orientation and reduce the "when does this end?" fatigue on long pages.

Medium Priority

Pricing tier differentiation could be clearer Assessment

Three pricing tiers (Starter $349, Professional $799, Enterprise) require cognitive work to compare. The difference between tiers is primarily "Chat" vs "Chat + Voice" — this could be made instantly clear with a simple feature comparison matrix rather than requiring users to read card descriptions.

High Priority

Journey has no intermediate capture point Assessment

The user journey is: read homepage → go to pricing → book demo or leave. There's no email capture, no "watch a video first", no "download the case study". Visitors who are interested but not ready to commit to a call have no way to engage further. This is a significant gap for B2B sales cycles where research happens over days/weeks.

Medium Priority

Pricing page doesn't reinforce value before asking for commitment Assessment

Visitors jump from the emotionally compelling homepage to a transactional pricing grid. The pricing page should open with a value reminder ("Stop losing $X/month to slow response") before showing prices. Anchoring against the cost of inaction makes $349/mo feel like a bargain.
