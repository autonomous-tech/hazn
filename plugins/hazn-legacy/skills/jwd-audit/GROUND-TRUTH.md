## 4. Component APIs (from frontend/src/components/ui/)

### Button (`@/components/ui/Button`)
- **Variants:** `primary` (vermillion bg), `primary-indigo`, `secondary` (border), `text` (vermillion + auto arrow), `dark` (ink bg)
- **Sizes:** `sm` (px-4 py-2), `md` (px-6 py-3), `lg` (px-8 py-4)
- **Base classes:** `font-medium inline-flex items-center justify-center leading-snug`
- **`text` variant** auto-appends `→` with hover translate — NEVER duplicate the arrow
- **Renders as** `<Link>` when `href` provided, `<button>` otherwise
- **Detection rule:** Any `<Link>` or `<a>` with `bg-vermillion`, `rounded-full`, button-like padding = 🔴 must use `<Button>`

### EyebrowLabel (`@/components/ui/EyebrowLabel`)
- **Color union:** `vermillion` | `caption` | `white` | `gold` | `ocean` | `indigo`
- **CSS class applied:** `eyebrow` (JetBrains Mono, 14px, 600, uppercase, tracking 0.1em)
- **Scope rule:** Section-level labels (before h2, at section top) → MUST use `<EyebrowLabel>`. Card-internal labels → raw `<span>`.
- **Detection rule:** Any `<span>` with `font-mono uppercase tracking-wider` outside a card = 🔴 must use `<EyebrowLabel>`

### stripCtaArrow (`@/lib/cta-utils`)
- **Strips:** `→`, `->`, `&rarr;`, `&#8594;`, `&#x2192;`, `➜`, `➝`, `➞`, `⟶` (trailing + whitespace)
- **Rule:** ALL CMS CTA text (`cta_text`, `primary_cta_text`, `secondary_cta_text`, `footer_cta_text`, `hero_cta_*_text`) MUST be wrapped in `stripCtaArrow()` before rendering
- **Detection rule:** Any `{...cta_text}` or `{...cta_label}` rendered without `stripCtaArrow()` = 🔴

### getBgClasses (`@/lib/bg-styles`)
- **Maps:** `dark` → `bg-ink text-white text-white/60`, `parchment` → `bg-parchment text-ink text-prose`, `white`/default → `bg-white text-ink text-prose`
- **Rule:** When `bg_style` is available from CMS, use `getBgClasses()` instead of hardcoding bg+text combos

## 6. Responsive Rules (from codebase patterns)

### Asymmetric Layout Trio (hero with image, 2-col content+visual)
All THREE required — missing any ONE = 🟡:
1. `text-center lg:text-left` — on headings/body text OR parent wrapper
2. `mx-auto lg:mx-0` — on content container
3. `justify-center lg:justify-start` — on CTA group flex wrapper

### Symmetric/Centered Layout
- `text-center` on header group
- `justify-center` on CTA groups

### Hero Image Visibility
- Side/background hero images: `hidden lg:block` wrapper (not shown on mobile)
- Decorative elements: `pointer-events-none`, parent `overflow-hidden`

### Badge/Tag Overflow Safety
- Inline badges in flex rows: `whitespace-nowrap` + `flex-shrink-0`

## 7. CTA Arrow Convention (from 10+ fix commits)

| Layer | Rule |
|-------|------|
| CMS Storage | Plain label text — NO arrows in any CTA field |
| Seed Data | No `\u2192` in CTA fields in `seed_remaining_pages.py` |
| Template Rendering | Wrap CMS CTA text in `stripCtaArrow()` (defense-in-depth) |
| Arrow Display | SVG `<svg>` elements only — NEVER text `→` in JSX |
| Button `text` variant | Auto-appends arrow — don't duplicate |
| Inline `.replace()` | FORBIDDEN — use shared `stripCtaArrow()` |
