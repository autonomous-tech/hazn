# Autonomous Tech — Deployment Repos & Publishing Workflow

> Shared knowledge for all agents. Hazn is the primary owner of this workflow,
> but any agent generating deliverables (proposals, audits, wireframes, landing pages)
> should follow this.
>
> ⚠️ Corrected 2026-03-03: landing-pages is PUBLIC (no auth). Earlier version had hallucinated auth details.

---

## Decision Rule — Always Ask First

**Before deploying anything, confirm:**
> "Should this be publicly accessible or internal only (shared via expiring link)?"

| Use case | Repo |
|---|---|
| Internal deliverables — confidential, shared via expiring link | `autonomous-proposals` → `docs.autonomoustech.ca` |
| Public-facing pages — anyone can access via URL | `landing-pages` → `pages.autonomoustech.ca` |

> ⚠️ **Audits always go to `autonomous-proposals` → `docs.autonomoustech.ca`** — never to `landing-pages`.
> Audit reports contain client analysis and competitive insights. Always treat as confidential.
> Share externally using the 30-day expiry share button.

## Repo: autonomous-proposals (PRIVATE — auth gated)
- **Repo:** `autonomous-tech/autonomous-proposals` (private)
- **Local clone:** `/home/rizki/autonomous-proposals`
- **Domain:** `docs.autonomoustech.ca` — Cloudflare Access, `@autonomoustech.ca` Google accounts only
- **Domain config:** Set in Cloudflare dashboard (not via CNAME file — the CNAME file in repo is a stale artifact, ignore it)
- **External sharing:** Share button generates 30-day expiry links via `share.autonomoustech.ca`
- **Deploy:** Push to `main` → GitHub Action → Cloudflare Wrangler (~30-60s)
- **Structure:** Client folders at repo root
  ```
  {client-name}/
  ├── index.html      ← required hub/nav page
  ├── *.html
  └── images/
  ```
- **Share button:** Must be in every HTML page before `</body>`. Copy from `briar-creek-construction/index.html`. Only renders when `CF_Authorization` cookie is present.

## Repo: landing-pages (PUBLIC)
- **Repo:** `autonomous-tech/landing-pages` (private repo, public deployment)
- **Local clone:** `/home/rizki/landing-pages`
- **Domain:** `pages.autonomoustech.ca` — fully public, no auth required
- **Deploy:** Push to `main` → Cloudflare Pages GitHub integration (~30-60s)
- **⚠️ NEVER edit `index.html` manually** — auto-generated by GitHub Action on every `.html` push
- **Structure:** Fixed category subfolders
  ```
  proposals/      ← client proposals
  audits/         ← audit reports
  docs/           ← internal docs, multi-file projects
  demos/          ← demos and prototypes
  lunch-n-learn/  ← presentations
  services/       ← service pages
  assets/         ← shared assets (logos, fonts)
  ```
- **Single-file pages:** `proposals/client-name-YYYY-MM-DD.html`
- **Multi-file pages:** `docs/project-name-YYYY-MM-DD/index.html` + assets in same folder
- **File naming:** lowercase, hyphenated, date-stamped
- **Logo paths:** `../logo-01.jpg` from subfolders, `../../logo-01.jpg` from sub-subfolders

## Git Commit Convention
Format: `Agent [Name]: [Page Type] for [Client Name]`

Git identity (set per agent):
```
git config user.email "[agentname]@autonomoustech.ca"
git config user.name "[AgentName]"
```

---

## Deploy Steps (both repos)
1. Generate HTML file(s)
2. Place in correct folder/structure
3. `git add . && git commit -m "Agent [Name]: [Type] for [Client]"`
4. `git push origin main`
5. Wait ~30-60s for deployment
6. Verify at the live URL
