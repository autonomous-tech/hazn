    **3. Create `hazn_platform/hazn_platform/orchestrator/output_collector.py`:**

    Follow RESEARCH.md Pattern 3 design. Key implementation:

    - `ArtifactType(StrEnum)`: REPORT, FINDINGS, CODE, METADATA

    - `CollectedArtifact(BaseModel)`: artifact_type, content, structured_data, metadata (all Pydantic validated)

    - `OutputCollector` class:
      - `_FINDING_PATTERN`: regex for structured findings. Must be tolerant -- match bullet items with **Severity/Priority**: and **Description/Issue**: and **Evidence**: and **Recommendation**: fields. Use `re.MULTILINE | re.DOTALL`. Handle both `*` and `-` bullet markers. Handle both "Severity" and "Priority" labels, both "Description" and "Issue" labels.
      - `_CODE_ARTIFACT_PATTERN`: regex for ` ```artifact {lang} {filepath}\n{code}\n``` `. Use `re.DOTALL`.
      - `_SECTION_PATTERN`: regex for `## ` headings to extract section names.

      - `collect(agent_output: str) -> list[CollectedArtifact]`:
        1. Always produce Report artifact: `ArtifactType.REPORT`, content=agent_output, metadata=self._extract_metadata(agent_output)
        2. Try to extract findings from `## Findings` section (or similar heading). If found and non-empty, produce Findings artifact with structured_data={"findings": list_of_finding_dicts}
        3. Extract code artifacts via `_CODE_ARTIFACT_PATTERN`. For each match, produce Code artifact with metadata={"language": lang, "filepath": path}, content=code_body
        4. Always produce Metadata artifact with structured_data from `_extract_metadata()`

      - `_extract_findings(text: str) -> list[dict]`:
        Find the `## Findings` section (between ## Findings heading and next ## heading or end).
        Apply `_FINDING_PATTERN` regex on that section.
        Return list of dicts: `{"severity": match[0], "description": match[1].strip(), "evidence": match[2].strip(), "recommendation": match[3].strip()}`
        If regex finds nothing, try a fallback: look for bullet items with at least severity+description.
        Return empty list if nothing parseable (never raise).

      - `_extract_code_artifacts(text: str) -> list[CollectedArtifact]`:
        Apply `_CODE_ARTIFACT_PATTERN` regex on full text.
        For each match: create CollectedArtifact(type=CODE, content=code, metadata={"language": lang, "filepath": path})

      - `_extract_metadata(text: str) -> dict`:
        Return `{"word_count": len(text.split()), "section_count": count of ## headings, "sections": list of heading texts, "char_count": len(text)}`

    - The module must NOT import Django models. OutputCollector produces `list[CollectedArtifact]` (pure Pydantic). The caller (executor.py in Phase 9) handles saving to WorkflowPhaseOutput.
    - Log at INFO level: number of artifacts collected, types found.
    - Log at WARNING level: findings section found but no findings parseable (format deviation).
