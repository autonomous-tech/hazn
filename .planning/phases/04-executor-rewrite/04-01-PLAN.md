  <name>Task 2: Rewrite session.py for one-Letta-agent-per-client</name>
  <files>hazn_platform/hazn_platform/orchestrator/session.py</files>
  <action>
Rewrite `session.py` from scratch. The new WorkflowSession:

**Constructor `__init__(self, workflow_name, agency, end_client, triggered_by)`:**
- Creates WorkflowRun record (same as current)
- Creates MeteringCallback via `MeteringCallback.from_agency()`
- Stores agency, end_client references for later use
- Does NOT create HaznMemory yet (deferred to load_client_context)
- Stores `_client_context: str = ""` for assembled memory context
- Stores `_letta_agent_id: str | None = None`

**`load_client_context(self) -> str`:**
- Per user decision: one Letta agent per client (not per-agent-type per-client)
- Call `get_letta_client()` and look up or create agent named `client--{end_client.pk}`
- Use the pattern from RESEARCH Pattern 3:
  ```python
  client = get_letta_client()
  agent_name = f"client--{self._end_client.pk}"
  existing = client.agents.list(name=agent_name)
  if existing:
      self._letta_agent_id = existing[0].id
  else:
      agent = client.agents.create(
          name=agent_name,
          system="Hazn client memory agent",
          memory_blocks=[{"label": "active_client_context", "value": ""}],
          tags=[f"l3:{self._end_client.pk}"],
      )
      self._letta_agent_id = agent.id
  ```
- Create `HaznMemory(agent_id=self._letta_agent_id, l3_client_id=end_client.pk, l2_agency_id=agency.pk)`
- Call `memory.load_client_context()` to hydrate from Letta
- Store the assembled context string via `memory._assemble_context()` in `self._client_context`
- Return the context string (executor will inject into system prompts)
- Wrap in try/except: if Letta is down, log warning and return empty string (non-fatal)

**`get_client_context(self) -> str`:**
- Returns the cached `_client_context` string

**`checkpoint(self, phase_id: str | None = None)`:**
- Per user decision: memory checkpoints after each phase completes
- If `_memory` exists, call `_memory.checkpoint_sync()`
- Call `_metering.flush_to_db(phase_id=phase_id)`
- Update `workflow_run.save(update_fields=["last_activity_at"])`

**`end(self, findings_by_agent=None)`:**
- End memory session if `_memory` exists: `_memory.end_session(findings or [])`
- Flush metering, update WorkflowRun to COMPLETED with totals
- Same logic as current but simplified (single memory instance, not dict)

**`fail(self, error: str)`:**
- Failure memory sync if `_memory` exists: `_memory.failure_sync()`
- Flush metering, update WorkflowRun to FAILED with error_details
- Same logic as current

**Remove entirely:**
- `get_memory()` method (replaced by single `_memory` instance)
- `log_conflicts()` method (conflict detection is deleted)
- `is_timed_out()` method (timeout handled by Celery, not session)

**Properties:**
- `workflow_run` (same as current)
- `metering` -- expose `_metering` as property (executor needs it for per-agent cost tracking)
