**`build_prior_phase_section(completed_outputs: list[WorkflowPhaseOutput]) -> str`** (module-level function):
- Per user decision: inject as `## Prior Phase Results` section with summary paragraph + structured findings, NOT full raw output
- Per Claude's discretion (from CONTEXT.md): inject only direct dependency outputs (filter by phase.depends_on)
- Cap at 10 findings per phase to manage token budget (Pitfall 3)
- Format:
  ```
  ---
  ## Prior Phase Results

  ### Phase: {phase_id}
  **Summary:** {summary}

  **Key Findings:**
  - [{severity}] {description} -- Recommendation: {recommendation}
  ```
- Return empty string if no completed outputs

**`WorkflowExecutor.__init__(self, workflow, session, registry)`:**
- Store workflow, session, registry
- Build `_phase_map: dict[str, WorkflowPhaseSchema]` from workflow.phases
- Init `_completed_phases: set[str]` and `_skipped_phases: set[str]`
- Init `_phase_outputs: dict[str, WorkflowPhaseOutput]` for collecting outputs per phase
- Create `_output_collector = OutputCollector()`

**`async run(self) -> WorkflowRun`:**
1. Start session: `await sync_to_async(self._session.start)()`
2. Emit workflow_started SSE event
3. Initialize Langfuse trace (non-fatal): `await sync_to_async(start_workflow_trace)(workflow_run)`
4. Load client memory context once: `await sync_to_async(self._session.load_client_context)()` (non-fatal, wrapped in try/except)
5. Walk DAG waves via `get_execution_order(self._workflow)`
6. For each wave:
   - Filter phases to those in `_phase_map` and not in `_skipped_phases`
   - Skip phases whose `depends_on` includes any phase in `_skipped_phases` (Pitfall 6)
   - Execute via `asyncio.gather(*[self._execute_with_retry(phase) for phase in wave_phases], return_exceptions=True)`
   - Process results: required failure halts, optional failure adds to _skipped_phases
   - Checkpoint after each wave: `await sync_to_async(self._session.checkpoint)(phase_id=...)`
7. End session: `await sync_to_async(self._session.end)()`
8. Emit workflow_completed SSE event
9. Return workflow_run

**`async _execute_with_retry(self, phase) -> dict | None`:**
- Per user decision: required phase failure retries once with fresh agent context, then halts
- Try `_execute_phase(phase)`. On exception:
  - If not `phase.required`: re-raise (no retry for optional phases)
  - Log warning, retry once: `_execute_phase(phase)` again
  - If retry also fails: raise RuntimeError with both errors

**`async _execute_phase(self, phase) -> dict | None`:**
1. Skip if no agent (informational phase)
2. Emit phase_started SSE event
3. Start phase span in Langfuse (non-fatal)
4. Assemble system prompt via `assemble_prompt(agent_type=phase.agent, skills=phase.skills, client_context={...})`
5. Append client memory context from session: `self._session.get_client_context()`
6. Build prior phase section: get completed outputs for this phase's `depends_on`, call `build_prior_phase_section()`, append to system prompt
7. Build initial prompt: `f"Execute the {phase.name} phase for client {l3_client_id}"`
8. Get registry singleton: `from hazn_platform.orchestrator import apps as apps_module; from hazn_platform.orchestrator.tools import build_registry; registry = apps_module._REGISTRY_SINGLETON or build_registry()`
9. Call Agent SDK `query()` directly per RESEARCH Pattern 1:
   ```python
   options = ClaudeAgentOptions(
       system_prompt=system_prompt,
       mcp_servers={"hazn": registry.get_server()},
       allowed_tools=registry.get_allowed_tools(phase.tools),
       max_turns=phase.max_turns,
       permission_mode="bypassPermissions",
   )
   final_text = ""
   usage_data = {}
   async for message in query(prompt=initial_prompt, options=options):
       if isinstance(message, AssistantMessage):
           for block in message.content:
               if isinstance(block, TextBlock):
                   final_text = block.text
       elif isinstance(message, ResultMessage):
           if message.result:
               final_text = message.result
           usage_data = {
               "input_tokens": (message.usage or {}).get("input_tokens", 0),
               "output_tokens": (message.usage or {}).get("output_tokens", 0),
               "total_cost": message.total_cost_usd or 0.0,
               "num_turns": message.num_turns,
               "duration_ms": message.duration_ms,
           }
           if message.is_error:
               raise RuntimeError(f"Agent execution failed: {message.result or 'unknown error'}")
   ```
10. Record metering: `self._session.metering.on_llm_call(...)` with tokens from usage_data
11. Collect artifacts via OutputCollector
12. Handle delivery phases: if outputs contain "branded_html_report" (generalized per Research Open Question 1), parse JSON, validate AuditReportPayload, render HTML via `render_report()`
13. Store phase output as WorkflowPhaseOutput record with content (dict with findings, artifacts), summary, html_content, markdown_source
14. Cache the WorkflowPhaseOutput in `_phase_outputs[phase.id]`
15. Emit phase_completed SSE event
16. Return output dict

**Error handling in _execute_phase:**
- Wrap entire body in try/except
- On exception: emit phase_failed SSE event (non-fatal), then re-raise (retry logic is in _execute_with_retry)

**SSE event payloads** (per Pitfall 5: emit ONLY from executor, not from tasks.py):
- workflow_started: `{"run_id", "event": "workflow_started", "status": "running"}`
- workflow_completed: `{"run_id", "event": "workflow_completed", "status": "completed"}`
- workflow_failed: `{"run_id", "event": "workflow_failed", "status": "failed", "error": ...}`
- phase_started: `{"run_id", "phase_id", "phase_name", "event": "phase_started", "status": "running"}`
- phase_completed: `{"run_id", "phase_id", "phase_name", "event": "phase_completed", "status": "completed", "summary": ...}`
- phase_failed: `{"run_id", "phase_id", "phase_name", "event": "phase_failed", "status": "failed", "error": ...}`
