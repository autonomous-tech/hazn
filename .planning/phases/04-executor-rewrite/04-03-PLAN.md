**1. Rewrite tasks.py from scratch.** The new version is a thin Celery wrapper (~80 lines):

```python
"""Celery tasks for async workflow execution.

Public API
----------
* ``run_workflow.delay(workflow_name, agency_id, client_id, triggered_by)``
"""
from __future__ import annotations
import asyncio
import logging
from celery import shared_task
from celery.exceptions import SoftTimeLimitExceeded
try:
    from billiard.exceptions import SoftTimeLimitExceeded as BilliardSoftTimeLimitExceeded
except ImportError:
    BilliardSoftTimeLimitExceeded = SoftTimeLimitExceeded
logger = logging.getLogger(__name__)
```

**`run_workflow` Celery shared_task:**
- Same decorator: `@shared_task(bind=True, max_retries=0, time_limit=4*3600+300, soft_time_limit=4*3600)`
- Parameters: `self, workflow_name: str, agency_id: str, client_id: str, triggered_by: str`
- Body:
  1. Import models: `from hazn_platform.core.models import Agency, EndClient`
  2. Look up Agency and EndClient by pk
  3. Load workflow YAML: `from hazn_platform.orchestrator.workflow_parser import load_workflow; slug = workflow_name.lower().replace(" ", "-"); workflow = load_workflow(f"hazn/workflows/{slug}.yaml")`
  4. Create session: `from hazn_platform.orchestrator.session import WorkflowSession; session = WorkflowSession(workflow_name=workflow_name, agency=agency, end_client=end_client, triggered_by=triggered_by)`
  5. Store Celery task ID on WorkflowRun for correlation
  6. Get ToolRegistry singleton: `from hazn_platform.orchestrator import apps as apps_module; from hazn_platform.orchestrator.tools import build_registry; registry = apps_module._REGISTRY_SINGLETON or build_registry()`
  7. Run executor via async bridge (Pitfall 1 -- use new_event_loop, NOT asyncio.run):
     ```python
     from hazn_platform.orchestrator.executor import WorkflowExecutor
     loop = asyncio.new_event_loop()
     try:
         executor = WorkflowExecutor(workflow, session, registry)
         loop.run_until_complete(executor.run())
     finally:
         loop.close()
     ```
  8. Return `str(session.workflow_run.pk)`

- **Do NOT emit SSE events from tasks.py** -- per Pitfall 5, SSE events are emitted ONLY from executor.py. The old tasks.py had duplicate events; remove them.

- Exception handling:
  - `SoftTimeLimitExceeded`: call `session.fail("Workflow timed out after 4 hours")`, return run pk
  - General Exception: call `session.fail(str(exc)[:500])`, re-raise
