<objective>
Implement the session lifecycle methods on HaznMemory: checkpoint sync (every 10 turns), failure sync (crash recovery), session end (findings to Postgres + learnings to archival + context wipe), and a standalone write_finding method for individual finding writes. Add integration tests for context injection timing, cross-client isolation, and full lifecycle against real Letta.

Purpose: These methods complete the memory lifecycle that makes agents compound -- learnings accumulate across sessions via checkpoints, partial work is never lost on crashes, and clean session termination ensures data goes to the right places. The standalone write_finding method enables the MCP server to write individual findings without triggering a full session end. The isolation test is critical: agents must never see another client's data.

Output: Updated memory.py with lifecycle methods, unit tests for lifecycle, integration tests against real Letta + Postgres.
</objective>

    Then implement in `hazn_platform/hazn_platform/core/memory.py`:

    `record_turn()`:
    - Increment self._turn_counter
    - If self._turn_counter >= 10: call self.checkpoint_sync(), reset counter to 0

    `checkpoint_sync()`:
    - If not self._pending_learnings: return (no-op)
    - For each learning in self._pending_learnings: call self._write_craft_learning(learning)
    - Clear self._pending_learnings

    `failure_sync()`:
    - For each learning in self._pending_learnings:
      - Modify confidence: min(learning.confidence * 0.7, 0.7)
      - Add "partial_sync" to tags
      - Call self._write_craft_learning(modified_learning)
    - Clear self._pending_learnings

    `write_finding(finding: StructuredFinding)`:
    - Use a dispatcher dict mapping finding_type to (Model, json_field_name):
      - "keyword" -> (Keyword, "metadata")
      - "audit" -> (Audit, "findings")
      - "campaign" -> (Campaign, "config")
      - "decision" -> (Decision, "outcome")
    - Build provenance dict: {"_provenance": {"workflow_run_id": str(finding.workflow_run_id), "agent_type": finding.agent_type, "session_timestamp": finding.session_timestamp.isoformat()}}
    - Merge provenance into finding.data (or into the JSONField value): create the model record with finding.data fields spread into model fields, and set the JSONField to include both the original JSON content and the _provenance key.
    - If finding_type is unknown, log a warning but do not raise (graceful degradation)

    `end_session(findings: list[StructuredFinding])`:
    - For each finding: call self.write_finding(finding)
    - Call self.checkpoint_sync() to flush remaining learnings
    - Wipe active_client_context block: self._client.agents.blocks.update(agent_id=self._agent_id, block_label="active_client_context", value="")
    - Flag session as ended (optional: set self._session_ended = True to prevent further operations)

    IMPORTANT: Provenance goes into EXISTING JSONFields on the marketing models. No migration is needed.
    - Keyword has `metadata` JSONField -> store provenance here
    - Audit has `findings` JSONField -> store provenance here (merge with audit findings data)
    - Campaign has `config` JSONField -> store provenance here
    - Decision has `outcome` JSONField -> store provenance here

    Use a dispatcher dict pattern, not if/elif chains.
