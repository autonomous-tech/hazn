<objective>
Create the HaznMemory swap-safe abstraction, its type system, and the MemoryCorrection audit model. This plan establishes the core memory interface that all subsequent plans build on: context loading from Django models into Letta blocks, semantic search over archival memory with recency+confidence ranking, cross-client insight querying from Postgres, and programmatic memory correction with full audit trails.

Purpose: HaznMemory is the single point of contact with Letta -- if Letta is ever swapped, only this file changes. This plan delivers the foundational methods that the session lifecycle (Plan 02-02) and MCP server (Plan 02-03) depend on.

Output: memory_types.py (Pydantic models), memory.py (HaznMemory class), MemoryCorrection Django model with migration, unit tests.
</objective>
