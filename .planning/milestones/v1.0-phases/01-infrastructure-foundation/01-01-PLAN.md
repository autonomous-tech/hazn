<task type="auto">
  <name>Task 2: Create Vault auto-unseal script and Makefile, validate all services start healthy</name>
  <files>
    scripts/vault-init.sh,

  <action>
    1. Create `scripts/vault-init.sh` -- the auto-init and unseal script:
       - Wait for Vault to be reachable (retry loop checking /v1/sys/health or /v1/sys/seal-status)
       - Check if Vault is initialized via /v1/sys/init endpoint
       - If not initialized: POST to /v1/sys/init with secret_shares=1, secret_threshold=1 (dev simplicity)
       - Extract unseal key and root token from response
       - Write to `.vault-keys` file (UNSEAL_KEY=..., ROOT_TOKEN=...)
       - If sealed: read UNSEAL_KEY from .vault-keys, POST to /v1/sys/unseal
       - Enable KV v2 secrets engine at `secret/` path (idempotent -- ignore error if already mounted)
       - Print status messages throughout
       - Make the script executable (chmod +x)
       - Use `curl` for Vault API calls and `python3 -c` or `jq` for JSON parsing (prefer python3 since it is available in the Django container)

    2. Extend the Makefile (cookiecutter generates a base one):
       - `make up`: docker compose -f docker-compose.local.yml up -d, then wait for services, then make vault-init
       - `make down`: docker compose -f docker-compose.local.yml down
       - `make seed`: docker compose run --rm django python manage.py seed_dev_data (will be implemented in Plan 03)
       - `make reset-db`: down -v, up -d, wait, vault-init, migrate, seed
       - `make vault-init`: bash scripts/vault-init.sh
       - `make test`: docker compose run --rm django pytest
       - `make logs`: docker compose logs -f
       - `make migrate`: docker compose run --rm django python manage.py migrate
       - All targets should use COMPOSE_FILE variable for DRY

    3. Run `make up` (or equivalent docker compose up -d) and verify all services reach healthy state:
       - `docker compose ps` should show all services as "healthy" or "running"
       - Run vault-init.sh and confirm Vault is initialized and unsealed
       - Check `.vault-keys` file was created
       - Verify Letta is reachable: `curl -sf http://localhost:8283/v1/health`
       - Verify Postgres has both databases: `docker compose exec postgres psql -U <user> -l` should show hazn and letta databases
       - Verify pgvector extension in letta database: `docker compose exec postgres psql -U <user> -d letta -c "SELECT extname FROM pg_extension WHERE extname='vector'"`
       - Verify Django responds (check the port it runs on)
       - Verify Next.js responds: `curl -sf http://localhost:3000`

    If any service fails to start, debug and fix the compose configuration before proceeding.
  </action>
