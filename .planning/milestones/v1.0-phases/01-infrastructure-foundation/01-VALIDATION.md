# Phase 1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest + pytest-django (Cookiecutter Django default) |
| **Config file** | `pytest.ini` or `setup.cfg` (generated by Cookiecutter Django) |
| **Quick run command** | `docker compose -f docker-compose.local.yml run --rm django pytest -x` |
| **Full suite command** | `docker compose -f docker-compose.local.yml run --rm django pytest` |
| **Estimated runtime** | ~30 seconds |

---

## Sampling Rate

- **After every task commit:** Run `docker compose -f docker-compose.local.yml run --rm django pytest -x`
- **After every plan wave:** Run `docker compose -f docker-compose.local.yml run --rm django pytest` + `docker compose ps` health check
- **Before `/gsd:verify-work`:** Full suite must be green + all 4 success criteria verified
- **Max feedback latency:** 30 seconds
