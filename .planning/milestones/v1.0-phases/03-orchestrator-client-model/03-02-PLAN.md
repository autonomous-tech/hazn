```python
class HaznMemory:
    def __init__(self, agent_id: str, end_client_id: str):
        """Per-session instance wrapping Letta access."""

    def load_client_context(self) -> str:
        """Load L2+L3 data from Postgres, inject into Letta block. Returns context string."""

    def record_turn(self) -> None:
        """Increment turn counter. Auto-triggers checkpoint_sync() at 10 turns."""

    def checkpoint_sync(self) -> int:
        """Flush pending learnings to Letta archival. Returns count flushed."""

    def failure_sync(self) -> int:
        """Preserve partial learnings on crash with reduced confidence."""

    def end_session(self, findings: list[StructuredFinding]) -> dict:
        """Write findings to Postgres, flush learnings, wipe context block."""

    def search_memory(self, query: str, limit: int = 10) -> list[dict]:
        """Semantic search of agent's archival memory."""
```

    Create `metering.py`:

    **class MeteringCallback:**
    - __init__(self, workflow_run_id: UUID, max_turns: int = 50, max_cost: float = 5.0, on_threshold_alert: Callable | None = None)
    - _agent_meters: dict[str, dict] tracking {tokens, cost, turns} per agent_id
    - on_llm_call(self, agent_id: str, tokens: int, cost: float): accumulate and check thresholds
    - flush_to_db(self, phase_id: str | None = None): write to WorkflowAgent via update_or_create
    - get_totals(self) -> dict: return aggregated totals across all agents

    Read max_turns and max_cost from agency.tool_preferences if available (keys: "max_turns", "max_cost_per_run"), falling back to defaults (50 turns, $5).

    Create `session.py`:

    **class WorkflowSession:**
    - __init__(self, workflow_name: str, agency: Agency, end_client: EndClient, triggered_by: str)
    - Creates WorkflowRun record (status=pending)
    - Initializes MeteringCallback with agency thresholds
    - workflow_run: WorkflowRun property
    - memories: dict[str, HaznMemory] -- one per agent

    - start(self) -> WorkflowRun:
      - Update status from "pending" to "running", set started_at
      - No context loading here -- context is loaded per-agent in executor

    - get_memory(self, agent_id: str, end_client_id: str) -> HaznMemory:
      - Lazy-create HaznMemory per agent (not per session -- different agents may have different contexts)

    - record_turn(self, agent_id: str):
      - Delegate to corresponding HaznMemory.record_turn()
      - Update WorkflowRun.turn_count and last_activity_at

    - checkpoint(self, phase_id: str | None = None):
      - Call checkpoint_sync on all active memories
      - Call metering.flush_to_db(phase_id)
      - Update WorkflowRun.last_activity_at

    - end(self, findings_by_agent: dict[str, list] | None = None) -> WorkflowRun:
      - Call end_session on each HaznMemory (with findings if provided)
      - Call metering.flush_to_db()
      - Update WorkflowRun: status=completed, ended_at=now, total_tokens/total_cost from metering totals
      - Return updated WorkflowRun

    - fail(self, error: str) -> WorkflowRun:
      - Call failure_sync on each HaznMemory
      - Call metering.flush_to_db()
      - Update WorkflowRun: status=failed, ended_at=now, error_details={"error": error}
      - Return updated WorkflowRun

    - is_timed_out(self, timeout_hours: int = 4) -> bool:
      - Check if last_activity_at is older than timeout_hours

    - log_conflicts(self, conflicts: list[dict]):
      - Append to WorkflowRun.conflict_log

    Per CONTEXT.md: credential fetching uses existing read_secret() via Vault AppRole (CRED-01). Raw secrets never appear in agent context (CRED-02) -- WorkflowSession MUST NOT store Vault secret values in WorkflowRun fields (status, error_details, conflict_log) or pass them into HaznMemory context blocks. Credentials are passed directly to tool calls only via get_credentials MCP tool. Credentials are scoped per L2/L3 via VaultCredential model (CRED-03). get_credentials MCP tool handles passing secrets to tool calls (CRED-04 -- already implemented in Phase 2).
