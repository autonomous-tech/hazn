CONTEXT.md locked decisions on conflict detection:
- locked_rules structured as JSON in Agency.house_style: {"locked_rules": [{"rule": "text", "category": "tone|terms|legal", "severity": "hard|soft"}]}
- hard rules always override L3
- soft rules: L3 wins by default, logged as info
- Conflict detection runs once at session start (pre-flight)
- Auto-resolved conflicts included in HITL queue as non-blocking info items

CONTEXT.md locked decisions on HITL:
- Four trigger types: conflict, checkpoint, cost_alert, uncertainty
- Timeout defaults: conflicts -> L3 wins; checkpoints -> auto-approve; cost alerts -> halt; uncertainty -> proceed with warning
- HITL items block by default; agencies can mark trigger types as non-blocking

    - Test: process_expired_items finds items past their timeout and applies timeout_action

    - Test: process_expired_items with timeout_action "proceed_with_warning" sets status to auto_resolved
    - Test: get_pending_items_for_run returns only pending HITLItems for a given WorkflowRun
    - Test: get_blocking_items_for_run returns only pending + blocking HITLItems
    - Test: has_blocking_items returns True when unresolved blocking items exist
  </behavior>
  <action>
    Create `hitl.py` with HITL queue management functions:

    **DEFAULT_TIMEOUT_ACTIONS** dict mapping trigger_type to default timeout_action:
    - "conflict" -> "l3_wins"
    - "checkpoint" -> "auto_approve"
    - "cost_alert" -> "halt"
    - "uncertainty" -> "proceed_with_warning"

    **create_hitl_item(workflow_run: WorkflowRun, trigger_type: str, title: str, details: dict, is_blocking: bool = True, timeout_hours: int = 24) -> HITLItem:**
    - Create HITLItem with trigger_type, title, details, is_blocking
    - Set timeout_action from DEFAULT_TIMEOUT_ACTIONS[trigger_type]
    - Override is_blocking if agency has configured non-blocking for this trigger type (check agency.tool_preferences.get("hitl_config", {}).get(trigger_type, {}).get("blocking", True))
    - Return created item

    **approve_hitl_item(item_id: UUID, resolved_by: str) -> HITLItem:**
    - Fetch item, validate status is "pending"
    - Update: status="approved", resolved_by, resolved_at=now
    - Return updated item

    **reject_hitl_item(item_id: UUID, resolved_by: str, reason: str = "") -> HITLItem:**
    - Fetch item, validate status is "pending"
    - Update: status="rejected", resolved_by, resolved_at=now, resolution={"reason": reason}
    - Return updated item

    **process_expired_items() -> list[HITLItem]:**
    - Query HITLItems where status="pending" and created_at + timeout_hours < now
    - For each expired item, apply timeout_action:
      - "l3_wins" or "auto_approve" or "proceed_with_warning" -> status="auto_resolved", resolution={"action": timeout_action, "auto": True}
      - "halt" -> status="timed_out", resolution={"action": "halt", "auto": True}
    - Return list of processed items

    **get_pending_items_for_run(workflow_run_id: UUID) -> QuerySet:**
    - Filter HITLItems by workflow_run_id and status="pending"

    **get_blocking_items_for_run(workflow_run_id: UUID) -> QuerySet:**
    - Filter by workflow_run_id, status="pending", is_blocking=True

    **has_blocking_items(workflow_run_id: UUID) -> bool:**
    - Return get_blocking_items_for_run(workflow_run_id).exists()
  </action>
