    Modify `hazn_platform/orchestrator/executor.py`:

       ```python
       from hazn_platform.qa.runner import should_run_qa, create_deliverable, handle_qa_result
       ```

    2. In `_execute_phase`, after the `WorkflowPhaseOutput.objects.create` call and before the return, add QA injection logic:
       ```python
       # Automatic QA injection (QA-01: runs on every deliverable)
       if should_run_qa(phase):
           try:
               phase_output_record = await sync_to_async(
                   WorkflowPhaseOutput.objects.filter(
                       workflow_run=workflow_run, phase_id=phase.id
                   ).first
               )()
               if phase_output_record:
                   deliverable = await sync_to_async(create_deliverable)(
                       workflow_run=workflow_run,
                       phase_output=phase_output_record,
                       phase=phase,
                   )
                   await sync_to_async(handle_qa_result)(
                       deliverable=deliverable,
                       workflow_run=workflow_run,
                       agency=workflow_run.agency,
                   )
           except Exception:
               # QA infrastructure crash: log but do NOT fail the phase
               # Per user decision: infrastructure issues should not block workflow
               logger.warning(
                   "QA injection failed for phase %s (infrastructure issue)",
                   phase.id,
                   exc_info=True,
               )
       ```

    3. The existing `run()` method already checks `has_blocking_items` after each wave, so QA FAIL verdicts that create blocking HITL items will naturally pause the workflow. No changes needed to `run()`.
